## SA-13 — Healthcheck ping URL no longer leaks its secret (fixed)

`src/upstreamwx/api/scheduler.py`

Healthchecks.io-style ping URLs carry a bearer secret in the path. The failure log emitted the full
`target` at DEBUG **with `exc_info`** — and a `requests` exception also stringifies with the full URL — so
both the message and the traceback leaked the token. New `_redact_ping_url()` logs only
`scheme://host/<redacted>` plus the **exception type name** (no `exc_info`, no exception string). Tested.

## SA-09 — Host validation (app) + a TLS deploy gate (fixed for what's in-repo)

`src/upstreamwx/config.py`, `api/app.py`, `deploy/deploy.sh`, `deploy/_lib.sh`, `deploy/config.env.example`,
`deploy/upstreamwx.env.example`

- **Application host validation.** New `api_trusted_hosts` (default **None** → off, so
  dev/CLI/tailnet/TestClient are unchanged). When a public host sets it, `TrustedHostMiddleware` is
  installed **outermost** — a bad `Host` is 400'd before the session/body middleware. The loopback names
  are always appended so the direct `/v1/health` probe and a local uvicorn keep working (Starlette strips
  the port before matching). `/v1/health.limits.trusted_hosts` echoes whether it's active. This matters
  because a direct hit on the loopback uvicorn previously answered for any `Host`.
- **Deploy TLS gate.** New `DEPLOY_REQUIRE_HTTPS` (default **0**). When `1` (public prod, set **after**
  certbot) `deploy.sh` fails the deploy unless the public endpoint serves HTTPS and plain HTTP redirects
  (301/308) to it. The **SA-01 session cookie is `Secure`**, so the access gate is inert without live
  HTTPS — this gate stops a public release silently shipping without TLS. Off by default so bootstrap /
  first deploy / tailnet (no cert yet) are unaffected.

## SA-06 — Reproducible install + no root execution of service-user code (fixed for what's in-repo)

`uv.lock` (new), `deploy/deploy.sh`, `deploy/bootstrap.sh`, `deploy/_lib.sh`

- **Committed `uv.lock`** (116 packages; `uv lock --check` clean, not git-ignored) — the reproducibility
  keystone. `deploy.sh` now installs with **`uv sync --frozen --no-dev`** (exact from the lock, prod-only,
  fails loudly if the lock is stale) as the service user, replacing the re-resolving `uv pip install -e`.
- **Removed the root execution of `.venv/bin/playwright install-deps`** — a root process executing a
  service-user-writable venv binary (the SA-06 trust-boundary crossing). Chromium's OS libraries already
  come from `bootstrap.sh`'s reviewed root-owned apt manifest; a new `_usable_chromium_present()` helper
  checks for a working browser **without running any venv binary**, and only then does the Google-Chrome-
  from-signed-apt fallback (root running apt is legitimate). Best-effort throughout — a missing browser
  only makes the PDF endpoint 503 (NFR-6).
- **Pinned the uv installer** to an exact version (`astral.sh/uv/${UV_VERSION}/install.sh`) instead of the
  always-latest installer piped into a root shell.

**Still open (deferred to a host validation pass — workplan §5):** root-owned atomic release directories +
symlink switch (the largest remaining SA-06 piece), a pinned/pre-staged Chromium revision, uv-installer
checksum verification, a version-controlled nginx `:443` block + `return 444` default server (SA-09 edge),
and SBOM/signed-tags/SHA-pinned Actions (SA-07).
